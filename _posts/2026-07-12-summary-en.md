## [World's First Invasive BCI Medical Device Approved in China](https://t.me/zaihuapd/42515) ⭐️ 10.0/10

China's National Medical Products Administration (NMPA) has approved the first invasive brain-computer interface (BCI) medical device, the Implantable Brain-Computer Interface Hand Function Compensation System by Boraycon Medical Technology (Shanghai), for clinical use. This marks the global debut of a commercial invasive BCI for hand function restoration in quadriplegic patients. This approval represents a paradigm shift in neurotechnology, moving invasive BCIs from research to clinical practice for paralysis treatment. It offers new hope for quadriplegic patients and sets a regulatory precedent that could accelerate development and approval of similar devices worldwide. The device uses an epidural minimally invasive implantation technique with wireless power supply and communication, paired with a pneumatic glove to assist hand grasping. Clinical trials showed significant improvement in hand grip function and quality of life for patients aged 18–60 with cervical spinal cord injury.

telegram · zaihuapd · Jul 12, 14:39

**Background**: Invasive brain-computer interfaces (BCIs) involve surgically implanting electrodes on or in the brain to record neural signals and translate them into commands for external devices. The epidural approach used here is considered semi-invasive, as electrodes are placed on the dura mater (outer brain covering) rather than penetrating the cortex, reducing surgical risk. Wireless power and communication enable the implant to operate without transdermal wires, while the pneumatic glove provides mechanical actuation for hand movement.

A systematic study reveals that Claude Code sends approximately 33,000 tokens in harness overhead before processing a user prompt, while OpenCode sends only about 7,000 tokens under similar conditions. This token inefficiency directly translates to higher costs for users and raises concerns about the opacity of Claude Code's system prompts and caching strategy, potentially influencing developer tooling choices. The study captured all requests between the coding agents and the Anthropic endpoint, measuring harness overhead including system blocks, tool schemas, and messages; Claude Code's overhead was 4.7x that of OpenCode.

**Background**: Agentic coding tools like Claude Code and OpenCode use a harness layer to orchestrate interactions with language models, which consumes tokens for system prompts and tool definitions. Token overhead refers to the tokens consumed by this harness before the actual user prompt is processed, contributing to total cost per task.

## [Grok CLI Uploads Entire Codebase by Default](https://gist.github.com/cereblab/dc9a40bc26120f4540e4e09b75ffb547) ⭐️ 8.0/10

Security researchers discovered that xAI's Grok CLI version 0.2.93 transmits entire codebases and sensitive files such as .env to Google Cloud Storage by default, even when the user disables the 'improve model' setting. The upload behavior cannot be deactivated. This flaw exposes developers' proprietary code and credentials to unauthorized servers, posing a severe privacy and security risk. It undermines trust in AI coding tools that handle sensitive data. The tool uses git bundle to package the entire repository and sends file contents along with conversational requests to Google Cloud Storage. In tests, a 12 GB repository uploaded over 5 GiB of data, and a file explicitly instructed not to be opened was still recoverable from the bundle.

**Background**: Grok CLI is a command-line coding agent powered by xAI's Grok models, designed to assist with development tasks. Git bundle is a command that packages all commits and references into a single file for transfer. The discovery highlights a lack of transparency in how AI tools handle user data.

## [EU to fine Big Tech for failing to protect consumers](https://www.ft.com/content/25640be5-a5bd-4548-81f9-bd0e16f87f35) ⭐️ 8.0/10

EU Justice Commissioner Michael McGrath announced that Brussels plans to grant itself new powers to fine large tech companies for failing to protect consumers, especially children, from online consumption traps and deceptive design patterns. This regulatory shift could significantly impact major tech firms by enforcing stricter consumer protection rules, potentially leading to substantial fines and forcing changes in interface design to eliminate dark patterns. The European Commission intends to propose legislation by the end of the year targeting addictive design, subscription traps, and other dark patterns; the new rules would also apply to smaller online merchants and game developers, not just large platforms.

**Background**: Dark patterns are user interfaces designed to trick users into actions they didn't intend, such as buying overpriced insurance or signing up for recurring subscriptions. Currently, consumer protection rules are enforced by member states, but McGrath noted that enforcement has never resulted in fines, failing to deter violations.
